# JATS XML → PDF 自动化映射与生成 — 技术方案说明书

## 一、项目概述

### 1.1 选题背景

本项目针对 IMR 学术期刊结构化技术创新大赛**选题二**：将符合 JATS（Journal Article Tag Suite）1.3 标准的 XML 文档自动转换为排版规范的 PDF 文件。

JATS 是美国国家信息标准协会（NISO）制定的学术期刊内容结构化标记标准，广泛应用于国际学术出版领域。然而，从 JATS XML 到最终出版 PDF 的转换过程长期依赖人工排版，效率低、成本高，且难以满足不同期刊的个性化排版需求。

### 1.2 项目目标

构建一套全自动化的 JATS XML → PDF 转换流水线，实现：
- 完整的 JATS 1.3 XML 语义解析
- 双栏学术排版（标题/摘要单栏，正文/参考文献双栏）
- 图表自动嵌入、编号与渲染
- 数学公式转换与渲染
- 参考文献自动格式化（Vancouver 风格）
- 输出可直接用于学术出版的 PDF 文档

## 二、技术路线

### 2.1 总体架构

```
JATS XML 文件
    │
    ▼
┌─────────────────┐
│  lxml 解析器     │  解析 JATS 1.3 XML，提取结构化数据
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  Python 数据模型 │  dataclass 类型安全的数据结构
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  Jinja2 模板引擎 │  渲染为完整 HTML 文档（含内嵌 CSS + base64 图片）
└──────┬──────────┘
       │
       ▼
┌─────────────────┐
│  PDF 渲染后端    │  Playwright Chromium / WeasyPrint / xhtml2pdf
└──────┬──────────┘
       │
       ▼
   PDF 输出文件
```

### 2.2 核心技术选型

| 组件 | 技术 | 选型理由 |
|------|------|----------|
| XML 解析 | lxml + XPath | 高性能，完美支持 JATS DTD 及命名空间 |
| 数据模型 | Python dataclasses | 类型安全，代码可读性强，便于维护 |
| 模板引擎 | Jinja2 | 灵活的递归模板，支持组件化复用 |
| 排版样式 | CSS Paged Media | 标准化布局语言，支持 @page、双栏、分页控制 |
| PDF 渲染（主） | Playwright Chromium | 完整 CSS3 支持（双栏、flexbox），无需 GTK3 依赖 |
| PDF 渲染（备） | WeasyPrint | CSS Paged Media 原生支持，命名页、页眉页脚 |
| 数学公式 | 自研 MathML→HTML 转换 | 轻量高效，纯 Python 无外部依赖 |
| 图片处理 | Pillow + Base64 | 自动压缩（≤1600px），内嵌 PDF 无外部依赖 |
| 命令行 | Click | 简洁的 CLI 接口 |

### 2.3 多后端策略

系统自动选择最佳 PDF 渲染后端：

1. **WeasyPrint**（优先）：CSS Paged Media 原生支持，排版质量最佳。需要 GTK3 运行库。
2. **Playwright Chromium**（推荐）：调用系统 Edge/Chrome 浏览器，完整 CSS3 支持（双栏、flexbox、列跨）。Windows 用户首选，无需安装 GTK3。
3. **xhtml2pdf**（兜底）：纯 Python 实现，无任何外部依赖，适合受限环境。

## 三、关键算法与创新点

### 3.1 递归段落-表格-图片顺序保持

传统逐标签解析会丢失 XML 中段、表、图的原始交织顺序。本方案在解析阶段记录 `children_in_order` 列表，模板渲染时按原始顺序输出，确保"正文提及某表→该表紧随出现"的学术排版规范。

### 3.2 模糊图片文件名匹配（创新点）

实际场景中 JATS XML 引用的图片名（如 `fig-01.jpg`）与 zip 包中实际文件名（如 `fig1.jpg`）经常不一致。本方案实现三层匹配策略：
1. **精确匹配**：路径 + 文件名完全一致
2. **递归文件名匹配**：忽略目录结构，按文件名查找
3. **模糊匹配**：去除连字符和下划线、去除前导零后比较（`fig-01` ↔ `fig1`）
4. **批量位置匹配**：当模糊匹配失败时，将未匹配的 href 与未使用的图片文件按数字排序后对应

### 3.3 超宽表格自动横排（创新点）

根据表格列数（含 colspan）自动判断是否需要横排页面：
- `<6 列`：在双栏内显示
- `≥8 列`：自动触发横排 A4 页面

横排通过多段 PDF 分别渲染 + PyPDF2 合并实现，配合连续页码打印。

### 3.4 MathML 纯 Python 转换（创新点）

不依赖 MathJax 等外部渲染引擎，自研 MathML → Unicode/HTML 转换器：
- 希腊字母映射表（α, β, γ, ...）
- 运算符 Unicode 替换（≤→≤, ×→×）
- `mfrac` 分式用 CSS flexbox 实现分数线
- `msub`/`msup`/`msubsup` 上下标支持
- 解析失败自动降级为纯文本提取

### 3.5 图片智能压缩

原始学术论文图片分辨率极高（可达 9990×6000px），直接嵌入导致 PDF 体积膨胀。本方案使用 Pillow 自动：
- 宽度超过 1600px 时等比缩放
- RGBA/P 模式转 RGB
- JPEG quality=85 压缩

将 1.1MB 原图压缩至 ~120KB，PDF 从 3MB 降至 ~2MB。

### 3.6 多源引用格式兼容（创新点）

JATS XML 中参考文献存在两种格式：
- `<element-citation>`：结构化字段（author, title, source, year...）
- `<mixed-citation>`：纯文本引用字符串

本方案同时兼容两种格式，结构化引用自动排版为 Vancouver 风格，纯文本引用原样保留。

## 六、工程规范

### 6.1 代码结构

- 模块化设计：parser / models / renderer / figure_resolver / math_handler / utils / cli 七个模块
- 所有模块均含 docstring 和类型注解
- 解析器单文件 ~650 行，渲染器 ~500 行
- Jinja2 模板组件化：8 个可复用组件

### 6.2 跨平台兼容

- Python 3.9+，纯 Python 实现
- Windows/Linux/macOS 均可运行
- 自动检测可用的 PDF 后端，无需手动配置

### 6.3 错误处理

- XML 解析异常友好提示
- 图片查找失败显示占位符（不影响整体输出）
- MathML 解析失败降级为纯文本
- PDF 渲染失败保留中间 HTML 供调试

## 七、总结与展望

本项目实现了一套完整的 JATS XML → PDF 自动化转换流水线，覆盖 JATS 1.3 标准的主要元素，支持双栏排版、图表渲染、数学公式、参考文献等学术出版核心需求。

未来可继续完善的方向：
1. 基于 Ghostscript 的 PDF 后压缩（进一步减小文件体积）
2. 更精细的横排表格判断（基于内容宽度而非仅列数）
3. 支持更多期刊自定义排版模板
4. 批量处理与 Web 界面
